The Time I Made this Project

I made this project during the 7th grade, when I was just beginning down my programming path. During this time, I was playing Pokemon Red Rescue Team on my gameboy, and I had an epiphany. The player character actually rarely moved in the game, it was the background that moved. Figuring out this mechanic, I made an entire proof of concept game I called "the labyrinth" that did just this.

I tried to keep this page as faithful to the original project as possible, so I made minimal changes to the source material. I kept the tacky background images and the terrible source code. Feel free to look at the code in my Github, but be warned. It is not pretty, and is very evident that a middle school student threw this together in his free time.

Features I First Implemented

This project in particular, though very poorly made, is still one of my proudest works to date, and that's because it incorporated so many small features that I had been toying with until this point. Below are just a few features I had implemented myself, however poorly.

1. View Port

As mentioned above, I had a sudden realization that the player character does not move, but the background. I hadn't learned about HTML Canvas yet, so I implemented this feature using div elements. I took advantage of CSS transforms and absolute positioning. I remember very vividly the struggles of positioning everything properly relative to its parent, but the effect once it worked out was incredibly rewarding.

2. Character by Character Text Display

This feature was an attempt at mimicking the feature of many rpg games where the text is slowly revealed when you talk to NPCs, like in Pokemon Ruby or Runescape (two games I had in mind at the time). I had recently learned about window.setTimeout(), so I thought it would be really cool to try and make this feature. I then made a mini "library" which I made for use in this project without worrying about the text.

Story

tl;dr There is no story. Get keys to open locked doors.

In this game, you are the blue circle in the center of the screen (graphic design was clearly not my strong suit). You are trying to collect keys to get through doors and get to the "stairs," similarly to Pokemon Mystery Dungeon. You may collect coins but that is completely optional.
